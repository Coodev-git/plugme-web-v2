🔁 Backend Flow

How the System Controls Every Job

PlugMe isn't just a UI. Every job moves through a controlled backend pipeline that enforces correct order, prevents skipping, and auto-triggers key actions.

📝
Job Created

Student posts request. Backend stores it, AI optimizes it, makes it visible to nearby workers.

💰
Offer Submitted

Workers submit bids with price + message. Backend links each offer to the job and notifies the student.

✅
Critical Point
Offer Accepted

Student accepts. Backend marks offer as accepted, assigns worker, updates status to In Progress, and auto-creates chat.

💬
Chat & Work

Both users coordinate through auto-created chat. Real-time location sharing enabled for meetups.

⭐
Completed & Rated

Job marked done. Payment released. Student rates worker. Campus Cred score updated.

🚨
Critical Backend Requirement

When a student accepts an offer, the system must automatically create a chat room and redirect both users to it. If this doesn't happen, the app is UI-only — not a working system. This single action is the core of what makes PlugMe functional.
